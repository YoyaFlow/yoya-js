# skill-creator

创建新的 Claude Code 技能（slash commands）。

## 触发条件

当用户要求创建新技能、添加 slash command 或创建自定义命令时触发此技能。

## 技能创建流程

### 1. 确定技能类型

首先询问用户或根据需求确定：

- **技能名称**：使用短横线命名（kebab-case），如 `my-skill`
- **技能描述**：一句话说明技能的用途
- **触发场景**：什么情况下应该使用此技能

### 2. 创建技能文件

在 `.claude/skills/` 目录下创建 `{skill-name}.md` 文件：

```markdown
# {skill-name}

{技能描述}

## 触发条件

{描述什么情况下触发此技能}

## 执行流程

1. {步骤 1}
2. {步骤 2}
3. {步骤 3}

## 输出格式

{描述技能执行后的输出或行为}

## 示例

{使用示例}
```

### 3. 注册技能（如需要）

如果技能需要在 `CLAUDE.md` 中注册，更新 `CLAUDE.md` 的"技能"部分：

```markdown
## 技能

- `{skill-name}` - {技能描述}
```

### 4. 验证技能

创建完成后告知用户：
- 技能文件路径
- 触发方式（使用 `/{skill-name}` 命令）
- 下一步建议

## 技能文件规范

### 命名规范
- 文件名：`{skill-name}.md`
- 技能名：小写字母 + 短横线（kebab-case）
- 标题：使用 `# {skill-name}`

### 内容结构
- **触发条件** - 明确说明何时触发
- **执行流程** - 分步骤说明要做什么
- **输出/行为** - 说明技能执行后的结果

### 最佳实践
- 保持技能职责单一，每个技能只做一件事
- 触发条件要具体，避免与其他技能冲突
- 执行流程要清晰、可操作
- 考虑与现有技能和项目的集成

## 示例：创建 build 技能

```markdown
# build

构建项目并验证输出。

## 触发条件

当用户要求构建项目、运行 build 命令或准备发布时触发。

## 执行流程

1. 运行 `npm run build` 执行构建
2. 检查构建输出是否有错误
3. 如有错误，分析原因并尝试修复
4. 构建成功后，报告输出文件位置和大小

## 输出格式

- 构建是否成功
- 输出文件路径
- 文件大小（如适用）
- 构建耗时（如适用）
```
